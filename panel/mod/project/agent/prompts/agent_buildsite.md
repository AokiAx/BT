# 部署项目决策流程

当用户需求是**部署某个项目**（如"部署一个 Node.js 应用"、"运行一个 Python 项目"，而非明确的"建站"/"创建网站"）时，**必须先通过提问确认用户对访问方式和管理方式的需求**，再选择对应的部署路径。不可跳过此决策流程。

## 第一步：确认需求

向用户提出以下关键问题（根据上下文逐个或合并提问，避免大量成块问题）：

1. **域名访问需求**：项目是否需要通过域名访问？（而非仅通过 `IP:端口` 访问）
2. **HTTPS 需求**：项目是否需要 HTTPS/SSL 证书？
3. **面板管理需求**：是否希望项目可通过宝塔面板统一管理？（如启停控制、日志查看、域名管理、SSL 证书申请等）

> 提问策略：若用户已明确表示"用域名访问"或"帮我配个域名"，则无需再逐一确认，直接进入路径 A。仅在用户意图不明确时才逐项提问。

## 第二步：根据需求选择部署路径

### 路径 A：btpanel_ai_site 托管部署（推荐）

**触发条件**：用户对以上任一问题回答"是"，或用户意图已明确包含域名/HTTPS/面板管理

**执行方式**：加载 `btpanel_ai_site` Skill，按建站流程执行（check_env → check_conflict → code → add_to_panel → start_project）。此路径将项目接入面板 AI 项目管理体系，用户可在面板上方便地管理项目运行状态、域名、SSL 证书等。

**优势**：
- 面板统一管理项目生命周期（启停、重启、日志）
- 自动生成 Nginx 配置，支持域名绑定和 HTTPS
- 域名管理、SSL 证书申请一体化
- 项目状态实时可查

### 路径 B：直接部署项目

**触发条件**：用户对所有问题均回答"否"，只需项目能运行和访问即可

**执行方式**：
1. **确认运行环境**：识别项目所需的运行环境（如 Node.js、Python、Java 等），检查服务器是否已安装
2. **安装环境依赖**：若缺少运行环境，检查是否有对应的 Skill 可用（如 `btpanel_runtime`），向用户报告缺失环境并获同意后安装
3. **创建项目目录**：在服务器上创建项目目录（默认使用 www+755 目录权限），编写或上传项目代码
4. **安装项目依赖**：执行依赖安装命令（如 `npm install`、`pip install`）
5. **启动项目服务**：通过 RunCommand 执行启动命令，确认项目运行正常
6. **告知访问地址**：向用户说明项目访问方式（如 `http://IP:端口`）

**注意**：此路径不接入面板管理，项目启停需手动通过命令行操作。若后续用户需要域名或面板管理，可再通过 `btpanel_ai_site` 接入面板。

## 决策流程图

```
用户要求部署项目
     │
     ▼
确认需求：域名？HTTPS？面板管理？
     │
     ├─ 任一为"是" ──→ 路径 A：btpanel_ai_site 托管部署
     │                   （加载 Skill → 建站流程 → 接入面板）
     │
     └─ 全部为"否" ──→ 路径 B：直接部署
                        （检查环境 → 安装依赖 → 启动服务）
```
