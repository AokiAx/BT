# 宝塔Nginx防火墙

## 1. 简介
宝塔Nginx防火墙（btWAF）是一款专为宝塔面板用户打造的深度集成型Web应用防火墙（WAF）。它基于Nginx核心构建，无需修改现有网络架构即可部署，旨在为网站提供全方位的安全防护。该防火墙能够有效抵御SQL注入、XSS跨站脚本、CC攻击、Webshell后门上传等常见网络威胁。通过可视化的管理界面，用户可以轻松配置防御策略，实时监控攻击态势，保障网站业务的高可用性与数据安全。它是构建网站纵深防御体系的重要第一道防线。

## 4. 规则优先级

当多个规则同时生效时，按以下优先级顺序执行（从高到低）：

```
IP白名单 > IP黑名单 > UA白名单 > UA黑名单 > URL关键词拦截 >
单URL CC防御 > URL白名单 > URL黑名单 > 蜘蛛池 > 禁止境外 >
禁止国内 > 地区限制 > 非浏览器拦截 > User-Agent > CC防御 >
Cookie > URI过滤 > URL参数 > Thinkphp防御 > POST > 网站自定义防御
```

**重要说明**：

- IP白名单优先级最高，命中后跳过所有拦截
- 网站自定义防御优先级最低
- 规则优先级决定了流量匹配和拦截的顺序

## 5. 全局配置

全局配置应用于所有站点，新添加的站点会自动继承全局设置作为默认值。

### 5.1 CC防御

防御CC攻击，具体防御参数可到站点配置中调整。

| 配置项       | 默认值     | 说明                     |
| ------------ | ---------- | ------------------------ |
| 模式         | 标准模式   | 宽松/标准/严格/超严格    |
| 验证级别     | 标准(推荐) | 访问时间：60 秒          |
| 访问次数     | 120 次     | ——                       |
| 封锁时间     | 7200 秒    | ——                       |
| URL级CC防御  | 开启       | ——                       |
| 单IP防御     | 关闭       | ——                       |
| 地区人机验证 | ——         | 选中地区自动开启增强模式 |

**规则示例**：单IP在 60 秒内累计请求同一 URL 超过 120 次，触发 CC 防御并封锁该 IP 7200 秒。

#### 防护模式说明

- **宽松模式**：以"URL + 参数"为统计标准，误报率极低
- **标准模式（推荐）**：以"URL"为统计标准（忽略参数），误报率较低
- **严格模式**：以"IP + UA"为统计标准，适合遭受针对性攻击时
- **超严格模式**：仅以"IP"为统计标准，仅建议极端攻击下临时使用

### 5.2 攻击次数拦截

封锁连续的恶意攻击请求，请到站点配置中调整阈值。

| 配置项   | 默认值   |
| -------- | -------- |
| 访问时间 | 120 秒   |
| 攻击次数 | 20 次    |
| 封锁时间 | 86400 秒 |

**规则示例**：120 秒内累计攻击超过 20 次，封锁 86400 秒。

### 5.4 黑白名单管理

#### IP 白名单

所有规则对白名单 IP 段无效（含 IP 黑名单、URL 黑名单）；IP 白名单优先级最高。

| 起始 IP      | 结束 IP      | 备注 | 操作 |
| ------------ | ------------ | ---- | ---- |
| 192.168.9.88 | 192.168.9.88 | ——   | 删除 |
| 127.0.0.1    | 127.0.0.255  | ——   | 删除 |

**添加示例**：

- 单 IP：`192.168.10.6` → 起始/结束均为 `192.168.10.6`
- 连续 IP：`192.168.100.6-100` → 起始 `192.168.100.6`，结束 `192.168.100.100`
- CIDR 段：`192.168.1.0/24` → 起始 `192.168.1.1`，结束 `192.168.1.254`

#### IP 黑名单

禁止访问的 IP，返回状态码 444。

#### UA 白名单

初始化阶段在 User-Agent 中匹配关键词，命中即跳过所有拦截（可作 API 密钥使用）。

#### UA 黑名单

初始化阶段在 User-Agent 中匹配关键词，命中即拦截。

#### URL 白名单

大部分规则对白名单 URL 无效（IP 黑名单仍生效）。

**标准模式（不带参数）**：

- 示例1：`/admin/admin.php?system=abc` → 添加 `/admin/admin.php`
- 示例2：`/admin/1`（1 可变 1-99）→ 添加 `/admin/[1-99]`

**标准模式（带参数）**：

- 示例 URL：`/index.php?id=1&pp=xx`
  - 写法1：`id=1`
  - 写法2：`id`（仅匹配参数名）
  - 写法3：`pp=xx`

#### URL 黑名单

禁止访问的 URL（IP/URL 白名单除外），返回状态码 403。

### 5.5 访问过滤

#### 非浏览器拦截

在被非浏览器攻击时开启，将应用于所有网站（有 CDN 的网站可能存在部分误报）。

#### HTTP 请求过滤

请求类型过滤 / 请求头过滤 / 语义分析开关。

**请求类型过滤**（默认全部放行，关闭按钮即拦截对应类型）：
POST / GET / PUT / OPTIONS / HEAD / DELETE / TRACE / PATCH / MOVE / COPY / LINK / UNLINK / WRAPPED / PROPFIND / PROPPATCH / MKCOL / CONNECT / SRARCH

**请求头过滤**（长度超限即拦截）：

| Header                    | 长度 | Header          | 长度  |
| ------------------------- | ---- | --------------- | ----- |
| host                      | 500  | user-agent      | 500   |
| connection                | 100  | accept          | 500   |
| content-length            | 100  | referer         | 10000 |
| cache-control             | 100  | accept-encoding | 500   |
| upgrade-insecure-requests | 100  | accept-language | 500   |
| origin                    | 500  | cookie          | 10000 |
| content-type              | 300  |                 |       |

**请求数量过滤**：

| 项目                       | 阈值         |
| -------------------------- | ------------ |
| 解析 base64 字符串最大长度 | 800,000 字节 |
| POST 最大参数个数          | 1,000 个     |
| GET 最大参数个数           | 1,000 个     |

#### 禁止国外/国内访问

- **禁止国外访问**：禁止中国以外地区访问站点
- **禁止国内访问**：禁止中国大陆（含港、澳、台）访问，状态码 444

#### URL 请求类型拦截

单独设置某 URL 允许的请求类型。

- 拦截：指定 URL **禁止**某些类型
- 只放行：指定 URL **仅允许**某些类型

#### API 接口防御

白名单内 API 仍可对特定接口启用 CC 防御。
正则写法：必须以 `^` 开头、`$` 结尾，例：`^/api/getuserinfo$`

#### 蜘蛛池

云端同步蜘蛛白名单，池内 IP 不触发拦截。
支持的蜘蛛：百度、谷歌、360、搜狗、雅虎、必应、头条、神马

## 10. 新版界面独有功能

### 10.1 流量限制

针对 API 接口或特定 URL 的请求速率控制。

#### 规则列表

| 规则名称 | 网站名 | URL  | 触发条件 | 执行操作 | 状态 | 操作 |
| -------- | ------ | ---- | -------- | -------- | ---- | ---- |

#### 配置说明

- **优先级**：流量限制的优先级是最低的
- **适用场景**：一般用于 API 接口，不建议全站开启
- **建议返回类型**：503 或 502
- **配置文件路径**：`/www/server/btwaf/html` (limit.html 和 limit.json)，修改后需重启 Nginx

#### 添加流量限制

**基本配置**

- 名称：请输入规则名
- 应用到网站：选择应用站点

**触发条件**

- 路径：指定 URL

**访问限制策略**

- 单个IP
- IP + UA
- 所有IP

**限制阈值**

- 每秒访问限制：[X] 次
- 时间访问限制：[Y] 秒内，超出 [Z] 次

**响应结果**

- 状态码 403/404/502/503
- 返回网页内容
- 封锁IP

### 10.2 增强自定义规则

新版界面提供更强大的自定义规则引擎。

#### 优先级

```
IP白名单 > IP黑名单 > UA白名单 > URL白名单 > UA黑名单 > URL黑名单 >
禁止海外 > 禁止国内 > 地区拦截 > 自定义拦截 > IDC拦截 > User-Agent拦截 > CC拦截
```

#### 规则匹配维度

- 客户端 IP 国家/地区
- 省份/直辖市/自治区
- 地级市/地区
- IP段
- 请求方式
- URI（不带参数）
- URI（带参数）
- URI 参数名称
- URI 请求参数
- 请求头
- POST 请求参数
- Post Body 内容匹配
- User Agent
- 引用方/referer

#### 执行操作

- 放行
- 拦截
- 人机验证

### 10.3 专属规则

针对特定站点或特定 URL 的精细化防护配置。

#### URL 专用过滤

为特定 URL 地址设置独立的过滤规则。

- **入口**：站点配置 > URL 专用过滤
- **功能**：指定 URL 路径 + 过滤规则正则表达式
- **继承性**：继承全局设置中已启用的规则

#### URL 增强模式

独立设置某个 URL 的验证规则。

- **功能**：针对特定 URL 开启人机验证、滑动验证等
- **优先级**：高于普通的 CC 防御规则

## 11. 操作指南

### 11.1 开启与基础设置

- **开启/关闭防护**：在"网站防护"列表中找到对应站点，点击"状态"开关
- **CDN适配**：若网站使用了 CDN，必须开启"CDN"选项，确保防火墙能从 HTTP 头中获取真实客户端 IP

### 11.2 CC防御配置建议

- **阈值设置**：建议"60秒内访问120次，封锁300秒"
- **模式选择**：一般使用标准模式，攻击时可临时开启严格模式
- **增强模式**：仅在普通模式无法防御时临时开启

### 11.3 规则与黑白名单管理

- **IP黑白名单**：支持单IP、IP段、CIDR格式
- **UA黑白名单**：支持完整字符串或关键词匹配
- **URL黑白名单**：使用正则匹配，不支持带参数
- **自定义规则**：支持多字段组合，AND/OR 逻辑

### 11.4 监控与日志

- **攻击报表**：选择时间段查看攻击统计与趋势
- **拦截日志**：输入IP查询具体攻击行为及拦截原因
- **误报处理**：确认误报后将相关IP或URL加入白名单

### 11.5 安全工具搭配建议

- **与系统防火墙搭配**：开启"四层拦截"，联动 iptables/firewalld
- **与CDN搭配**：CDN清洗流量，防火墙作为源站最后防线

### 11.6 配置导入/导出

支持对防火墙全局配置进行备份与迁移：

- **导出内容**：CC配置、IP黑白名单、UA黑白名单、URL黑白名单、敏感文字替换、URL关键词拦截等
- **注意**：仅导出全局配置，不包含各站点的独立配置

## 12. 常见问答

### Q: 为什么开启防火墙后，网站后台某些功能无法使用？

**A**: 这通常是触发了SQL注入或XSS防御规则（误报）。请查看"拦截日志"定位对应时间的记录，确认是误报后将相关URL或参数加入白名单。

### Q: 如何判断网站是否遭受了CC攻击？

**A**: 若网站访问变慢、出现502/504错误且CPU飙升，同时防火墙"概览"中拦截次数激增，极大概率是遭遇了CC攻击。请检查CC防御是否开启并适当调整阈值或开启增强模式。

### Q: 搜索引擎爬虫会被拦截吗？

**A**: 防火墙默认内置了主流搜索引擎（如Google、百度）的白名单（蜘蛛池）。只要爬虫IP在白名单内，即使开启"禁国外"或高强度防护，通常也会放行。

### Q: "四层拦截"和普通拦截有什么区别？

**A**: 普通拦截由Nginx处理，会消耗少量应用层资源；"四层拦截"则联动系统防火墙在网络层直接丢弃数据包，恶意流量不进入Web服务，性能损耗极低。

### Q: 网站开启了CDN，为什么防火墙拦截的是CDN节点的IP？

**A**: 这是因为未配置CDN兼容选项。请务必在防火墙的"网站设置"中开启"CDN"选项，以便防火墙从HTTP头（如X-Forwarded-For）中获取真实的客户端IP。

### Q: 开启防火墙会影响服务器性能吗？

**A**: 会有轻微的性能开销（通常增加1%-2%左右的CPU占用），相对于其提供的安全防护价值，这种损耗通常可以忽略不计。

### Q: 为什么开启了"禁止海外"还是能看到拦截记录？

**A**: 默认情况下，"禁止海外"直接拦截不记录日志以节省IO。如果开启了"禁国外增强封锁"或相关日志选项，则会记录拦截详情。

### Q: 开启Nginx防火墙之后占用会增大吗？

**A**: 会增加，但是只会增加1%-2%左右，整体影响不大。

### Q: 网站被CC攻击后，需要如何处理？
1. 点击左侧栏的【WAF】，再点击【全局设置】，看到CC防护，设置规则
2. cc推荐配置如下
```
60秒内访问120次，封锁7200秒
```

### Q: 网站被广东地区刷流量，需要如何限制？

1. 若在广东地区的业务比较少，可点击【WAF】-【地区限制】，添加广东省限制
2. 若在广东地区的业务比较多，
- 可点击【WAF】-【流量限制】，添加访问限制，推荐单个IP，每 20秒 访问超出 20 次，返回503
- 点击【自定义规则】，配置规则如下
```
匹配规则：省份/直辖市/自治区
匹配条件：等于/完全匹配
匹配内容：广东
执行操作
    操作选项：人机验证
    验证方式：无感验证
```

### Q: 如何针对特定URL开启CC防御？

1. 点击左侧栏【WAF】-【全局设置】，搜索“URL”；
2. 进入"URL增强模式"，为特定URL设置独立的验证规则；
3. 配置触发阈值和封锁时间


### Q: 如何拦截未授权登录admin登录界面？

1. 通过"自定义规则"配置
2. 匹配规则：URI包含 `/admin` 或 `/admin.php`
3. 匹配条件：请求次数超过阈值
4. 执行操作：拦截或人机验证
